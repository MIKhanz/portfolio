The Problem

Eight isolated systems with no shared data layer

Every system — ERP, offer tool, supply chain, ITSM, delivery tracker, notifications, logistics, and reporting — operated in complete isolation. Every handoff was manual, every status update an email. Configuration errors were discovered at delivery, not at intake.

  • Manual data entry duplicated across 5+ tools per order
  • Sequential approvals creating 3–5 day bottlenecks
  • CIQ and BoM errors found at delivery, not at intake
  • Order status tracked in spreadsheets and email threads
  • Finance and supply chain engaged too late in the cycle
  • Zero KPIs, SLA tracking, or continuous improvement loop
  • Tribal knowledge with no SOPs or audit trail
  • Subscription contracts created and tracked manually

My approach: integration-first, automation-driven

Rather than replacing systems, I designed an integration-first strategy — build an API middleware and master data layer connecting all existing tools, then layer automation and workflow governance on top.

  • Discovery sprint mapping all processes and dependencies before any build
  • Quick-win phase to prove value and build team confidence early
  • Event-driven architecture with shared master data repository
  • Rule-based auto-approval replacing sequential sign-off chains
  • CIQ/BoM engine embedded at offer creation, not delivery
  • Centralised Power BI cockpit as the single visibility layer
  • GIC code mapping to align product categories with financial reporting
  • Phased rollout with measurable KPIs at each milestone gate

05

Qualitative Impact

Parallel Approvals Replaced Sequential Bottleneck

Multi-day chains redesigned into parallel, rule-gated workflows. Standard orders auto-approve in seconds; complex orders run concurrent approval tracks with no unnecessary waiting.

Single Source of Truth Across All Teams

Replaced 8 disconnected data stores with a master data repository. Every system consumes from one canonical record — eliminating conflicts and the tribal knowledge that slowed onboarding.

Finance & Supply Chain Integrated Upstream

Moved credit validation and supply chain readiness to offer creation — not delivery. Finance and procurement now have real-time data at the point of commitment.

Real-Time Operational Visibility

Replaced spreadsheet-tracked order status with a live cockpit delivering KPIs, SLA adherence, delivery capacity, and invoice accuracy to all stakeholder groups simultaneously.
